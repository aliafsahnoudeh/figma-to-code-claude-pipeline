Set up the local development environment for TaskFlow. There is no Docker, no database, and no auth — the goal is to get the FastAPI backend, the Next.js frontend, and the Design System all building.

## Version requirements

| Tool    | Required | Notes                                 |
| ------- | -------- | ------------------------------------- |
| Python  | 3.13.x   | Backend uses Python 3.13              |
| Node.js | ≥ 20     | Required by web/ and design-system/   |
| pnpm    | ≥ 8      | Package manager for the workspaces    |
| uv      | latest   | Python package manager for `backend/` |

## Process

### 1. Detect OS and check prerequisites

```bash
uname -s         # Darwin = macOS, Linux = Linux
python3.13 --version
node --version
pnpm --version
uv --version
```

Report what's installed and what's missing. Install anything missing (only with user confirmation):

#### macOS (Homebrew)

```bash
brew install python@3.13 node@20
corepack enable && corepack prepare pnpm@latest --activate
curl -LsSf https://astral.sh/uv/install.sh | sh
```

#### Linux (Debian/Ubuntu)

```bash
sudo add-apt-repository ppa:deadsnakes/ppa
sudo apt-get update
sudo apt-get install -y python3.13 python3.13-venv python3.13-dev
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
corepack enable && corepack prepare pnpm@latest --activate
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 2. Install project dependencies

```bash
cd backend && uv sync
cd ../design-system && pnpm install && pnpm build:packages    # must build before web/
cd ../web && pnpm install
```

### 3. Configure Claude Code settings

From the repo root:

```bash
node scripts/init-claude.js
```

This reads `.claude/settings.example.json`, replaces `$PWD` with the absolute path, merges with any existing `.claude/settings.local.json` (preserving user-added permissions), and creates `.claude/dashboard/` symlinks pointing to the user's global Claude config and auto-memory directory.

If the user wants to use the Figma MCP, they must replace `YOUR_TOKEN` in `.claude/settings.local.json` with a Figma personal access token.

### 4. Verify the setup

```bash
cd backend && uv run pytest -q
cd ../design-system && pnpm validate:tokens
cd ../web && pnpm test
```

### 5. Run the dev servers (optional smoke test)

```bash
# Terminal 1 — backend
cd backend && ./scripts/run_local.sh
# Terminal 2 — frontend
cd web && pnpm dev
```

Then open http://localhost:3000 — you should see the projects dashboard.

## Success criteria

- All prerequisites installed and version-verified
- `uv sync`, `pnpm install`, `pnpm build:packages` all succeeded
- `.claude/settings.local.json` generated by `init-claude.js`
- Backend tests pass, frontend tests pass, token validation passes
- `localhost:3000` and `localhost:8000` both respond

## Troubleshooting

- **Port 3000 / 8000 already in use** — `lsof -i :3000` (or `:8000`) and kill the offender.
- **`pnpm` not found after install** — `corepack enable && hash -r`.
- **`uv` not found after install** — re-source `~/.zshrc` or `~/.bashrc`, or add `$HOME/.local/bin` to `PATH`.
- **`@taskflow/components` not found in `web/`** — run `pnpm build:packages` in `design-system/` before `web/`.
